# Review Checklist

## Backend

- API ルーティングとサービス層の責務が分離されているか
- 設定値が環境変数から読み込まれているか
- 例外時にフロントエンド側で扱える形に整理できているか
- 将来の実 RAG 実装へ差し替えしやすい構造になっているか

## Frontend

- API レスポンスの型定義が明示されているか
- ローディング状態とエラー状態が UI 上で判別できるか
- 回答、引用元、検索デバッグ情報が分かれて表示されているか
- API ベース URL がコード内にハードコードされすぎていないか

## Security

- シークレットを Git に含めていないか
- Day 1 時点で Azure 実キーが不要な構成になっているか
- フロントエンドに Azure キーを露出させていないか

## Documentation

- README に起動手順があるか
- アーキテクチャ説明が実装内容と一致しているか
- 知識文書の体裁が揃っているか
