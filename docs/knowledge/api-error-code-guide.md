# API エラーコードガイド

## 概要

主要 API の代表的なエラーコードと対応方針を示します。

## 前提条件

- エラー応答に `code` と `message` が含まれていること
- リクエスト ID をログで追跡できること

## 業務ルール

1. `400` は入力不備として扱います。
2. `401` は認証失敗またはセッション切れとして扱います。
3. `403` は権限不足として扱います。
4. `500` はサーバー障害として一次調査を開始します。

## 例外・注意事項

- 同一エラーが短時間で増加した場合は障害起票を優先します。
- ユーザー向けメッセージに内部例外をそのまま返却しません。

## FAQ

### Q. `429` が返る場合は何を確認しますか。

レート制限設定と短時間の集中アクセス有無を確認します。
