# メッセージ機能仕様

## 概要
学校ユーザーと求職者間のメッセージ機能の仕様を定義します。
応募後にのみメッセージが可能な設計です。

## メッセージ送受信ルール

### 送信可能な条件
- 学校ユーザー → 求職者: 当該求職者が自校の求人に応募済みであること
- 求職者 → 学校ユーザー: 当該学校の求人に応募済みであること
- 応募が取り消し・辞退された場合は新規送信不可（既存メッセージは閲覧のみ可）

### 送信制限
- 1回のメッセージ: 最大2000文字
- 添付ファイル: 1件あたり10MB以内、拡張子は pdf / doc / docx / jpg / png のみ
- 1日の送信上限: 1スレッドあたり50件

## メッセージスレッド管理

- 1応募 = 1スレッド（応募IDで紐付け）
- スレッド一覧は最終メッセージ日時の降順で表示
- 未読件数をバッジで表示

## 通知ルール

| イベント | 通知方法 |
|---------|---------|
| 新着メッセージ | メール通知 + アプリ内通知 |
| メッセージ既読 | アプリ内のみ（既読マーク更新） |

メール通知のタイミング:
- 受信から15分以内に既読されていない場合に送信
- 1日に同一スレッドから最大3件まで

## 禁止コンテンツ

以下を含むメッセージは送信不可:
- 外部連絡先（電話番号、メールアドレス、SNSアカウント）の記載
- 差別的表現
- 求人条件と異なる採用条件の提示
- URLリンク（ホワイトリスト外）

検知方法: 送信時に正規表現パターンチェック、違反の場合はエラー返却。

## システム自動メッセージ

以下のイベント発生時にシステムが自動でメッセージを送信:
- 応募受付完了
- 書類選考結果（通過 / 不通過）
- 面接日程確定
- 内定通知

自動メッセージはシステムアカウントから送信され、返信不可。

## データ保持

- メッセージ履歴は最終送信から1年間保持
- 詳細はdata-retention-policy.mdを参照

## エラーケース

| エラーコード | 説明 |
|------------|------|
| E_MSG_NO_THREAD | 応募のないスレッドへの送信 |
| E_MSG_DAILY_LIMIT | 1日の送信上限超過 |
| E_MSG_FORBIDDEN_CONTENT | 禁止コンテンツ検知 |
| E_MSG_FILE_TOO_LARGE | 添付ファイルサイズ超過 |
