# 求人検索・フィルター仕様

## 概要
求職者が求人一覧を検索・絞り込む際の仕様を定義します。

## 検索方式

### キーワード検索
- 対象フィールド: 求人タイトル、職種説明、学校名、勤務地
- 部分一致（前方/中間/後方すべて対応）
- 複数キーワードはスペース区切りでAND検索
- 検索文字数上限: 100文字

### フィルター条件

| フィルター | 型 | 説明 |
|----------|-----|------|
| 都道府県 | multiselect | 複数選択可、「全国」も選択可 |
| 職種カテゴリ | multiselect | 大カテゴリ → 小カテゴリの2段階 |
| 雇用形態 | multiselect | 正社員 / 契約社員 / パート / インターン |
| 給与範囲 | range | 月給・時給・年収から選択 |
| 掲載期間 | select | 本日 / 3日以内 / 1週間以内 / 1ヶ月以内 |
| 学歴条件 | multiselect | 高卒以上 / 専卒以上 / 大卒以上 / 不問 |
| リモートワーク | boolean | 可 / 不可 / 一部可 |

## ソート順

- デフォルト: 新着順（掲載日降順）
- 選択肢: 新着順 / 給与高い順 / マッチ度順（ログイン時のみ）

## ページネーション

- 1ページあたり20件
- 最大表示: 1000件（50ページ）
- 1000件を超える場合は「条件を絞り込んでください」と表示

## 検索結果の表示項目

各求人カードに表示する項目:
1. 学校名（ロゴ付き）
2. 求人タイトル
3. 勤務地
4. 雇用形態
5. 給与（概要）
6. 掲載日
7. 応募状況バッジ（募集中 / 残りわずか / 締切間近）

## 業務ルール

1. 掲載停止中の求人は検索結果に表示しない
2. 掲載期限が本日で切れる求人には「締切間近」バッジを表示
3. ログイン済み求職者には既応募求人に「応募済み」バッジを表示
4. 検索ログはユーザーごとに30日間保持（マッチング改善に利用）

## パフォーマンス要件

- 検索レスポンス: 2秒以内
- キャッシュ: 同一条件の検索結果を5分間キャッシュ
