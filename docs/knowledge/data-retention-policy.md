# データ保持・削除ポリシー

## 概要
本ドキュメントでは、各データ種別のシステム内での保持期間と削除ルールを定めます。

## データ種別ごとの保持期間

| データ種別 | 保持期間 | 削除方式 | 備考 |
|-----------|---------|---------|------|
| 求職者アカウント | 退会後3年 | 論理削除 → 物理削除 | 法的義務のため |
| 学校ユーザーアカウント | 契約終了後1年 | 論理削除 → 物理削除 | |
| 求人情報 | 掲載終了後2年 | 論理削除 | 統計分析用 |
| 応募記録 | 採否確定後3年 | 論理削除 | 雇用法令対応 |
| メッセージ履歴 | 最終送信から1年 | 物理削除 | |
| ログイン履歴 | 90日 | 自動物理削除 | バッチで毎日削除 |
| 検索ログ | 30日 | 自動物理削除 | |
| 操作ログ（管理者） | 1年 | 物理削除 | |
| 添付ファイル（履歴書等） | 退会後1年 | Blob削除 | Azureストレージから削除 |
| CSVエクスポートファイル | 生成から7日 | 自動物理削除 | |

## 論理削除・物理削除の定義

### 論理削除
- `is_deleted = true` および `deleted_at = <タイムスタンプ>` を設定
- 通常のクエリからは除外される
- データは引き続きDBに存在する

### 物理削除
- DBレコードを完全に削除
- 復旧不可能

## 退会・解約処理フロー

### 求職者退会
1. 退会申請受付
2. `is_deleted = true` に更新（論理削除）
3. 個人識別情報をマスキング（氏名、メール等を匿名化）
4. 3年後にバッチで物理削除

### 学校契約終了
1. 管理者が契約終了処理を実施
2. 学校ユーザーを全員論理削除
3. 公開中の求人を全件掲載停止
4. 1年後にバッチで物理削除

## 自動削除バッチ

- 実行タイミング: 毎日 AM 3:00
- 対象: 保持期間を超えたレコード
- ログ出力: 削除件数を application log に記録
- 失敗時: アラートを管理者にメール送信

## ユーザーからの削除依頼（GDPR / 個人情報保護法対応）

- 求職者は設定画面から「アカウント削除」を申請可能
- 申請から14日以内に処理完了
- 法的保持義務があるデータは即時削除対象外（応募記録等）
- 削除完了通知メールを送信
