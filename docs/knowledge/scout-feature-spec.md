# スカウト機能仕様

## 概要
学校ユーザーが求職者に対してスカウトメッセージを送信する機能の仕様を定義します。

## スカウトとは
学校側から求職者に対してアプローチできる機能です。
求職者の公開プロフィールを閲覧し、興味を持った場合にスカウトを送信できます。

## 送信可能条件

- 学校ユーザーであること
- スカウト送信数が月次上限内であること（プランによる）
- 対象求職者がスカウト受信を「許可」に設定していること
- 同一求職者への月内重複送信は不可

## スカウト送信フロー

1. 求職者プロフィールを閲覧
2. 「スカウトを送る」ボタンを押下
3. 対象求人を選択（必須）
4. スカウトメッセージを入力（50〜1000文字）
5. 送信確認ダイアログ
6. 送信完了 → 求職者に通知

## スカウトメッセージの構成

必須記載事項:
- 学校名（自動挿入）
- 対象求人タイトル（選択した求人から自動挿入）
- 担当者名

任意記載事項:
- スカウト理由
- 求人のアピールポイント

## 求職者側の受信設定

設定画面でスカウト受信可否を選択可能:
- **全て受信**: すべての学校からのスカウトを受け取る
- **受信しない**: スカウトを受け取らない

※デフォルトは「全て受信」

## 求職者側の対応

スカウト受信後に選択できるアクション:
- **応募する**: スカウトに対してそのまま応募
- **メッセージを返す**: 返信メッセージを送信（応募前の質問等）
- **スキップ**: 特に返答しない（学校には通知されない）

## 月次上限のリセット

- 毎月1日 AM 0:00 にリセット
- リセット前日にダッシュボードに残数を表示
- 残数が少なくなった場合（上限の20%以下）に警告バナーを表示

## 統計・分析

学校ダッシュボードで以下を確認可能:
- 月次スカウト送信数 / 残数
- スカウト開封率（求職者がスカウトを開封した割合）
- スカウト応募転換率（スカウトから応募した割合）
