# 管理者ユーザー管理機能

## 概要
本ドキュメントでは、管理者が行うユーザー（学校・求職者）管理の操作と権限を説明します。

## 管理者の種別

| 種別 | 説明 | 権限範囲 |
|------|------|---------|
| システム管理者 | 全機能にアクセス可能 | 全ユーザー・全設定 |
| 運用管理者 | 日常運用業務を担当 | ユーザー管理・コンテンツ管理 |
| 閲覧専用管理者 | 統計・レポート閲覧のみ | 閲覧のみ |

## ユーザー管理メニュー

### 学校ユーザー管理
- 学校アカウントの新規作成
- 学校情報の編集（学校名、住所、担当者）
- アカウントの有効化 / 停止
- 契約プランの変更
- ログイン履歴の確認
- 代理ログイン（動作確認用）

### 求職者管理
- 求職者一覧の検索・閲覧
- プロフィールの閲覧
- アカウントの停止（不正利用対応）
- 退会処理の実施
- 本人確認書類の確認

## アカウント停止フロー

1. 管理者が対象ユーザーを検索
2. 「アカウント停止」ボタンを押下
3. 停止理由をプルダウンから選択（必須）
4. 確認ダイアログで承認
5. `is_active = false` に更新
6. 対象ユーザーに停止通知メール送信
7. 操作ログに記録（管理者ID・日時・理由）

### 停止理由コード

| コード | 説明 |
|--------|------|
| FRAUD | 不正利用の疑い |
| SPAM | スパム行為 |
| REPORT | 他ユーザーからの報告 |
| CONTRACT_END | 契約終了 |
| REQUEST | 本人からの申し出 |
| OTHER | その他 |

## 代理ログイン

- 対象: 学校ユーザーのみ（求職者への代理ログインは不可）
- 利用目的: 動作確認・サポート対応
- 制限: セッション有効期限30分、操作はすべて閲覧のみ
- ログ: 代理ログイン開始・終了・操作内容を記録

## 監査ログ

管理者の操作は全件ログに記録:
- 操作日時
- 管理者ID・種別
- 操作種別
- 対象ユーザーID
- 変更前後の値

ログ保持期間: 1年（data-retention-policy.md参照）

## セキュリティ要件

- 管理者画面は社内ネットワーク（または VPN）からのみアクセス可
- MFA（多要素認証）必須
- セッションタイムアウト: 30分
- パスワード要件: 12文字以上、英大文字・小文字・数字・記号を各1文字以上含む
