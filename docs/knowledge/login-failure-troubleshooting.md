# ログイン失敗時の確認手順

## 概要

ログインできない問い合わせに対する一次切り分け手順です。

## 前提条件

- 問い合わせ元ユーザーの ID もしくはメールアドレスが確認できること
- 障害受付番号が採番されていること

## 業務ルール

1. アカウント状態が有効か確認します。
2. ロック状態の場合は解除手順を案内します。
3. パスワード再設定メールの送信履歴を確認します。
4. SSO 利用時は連携先 IdP の障害有無も確認します。

## 例外・注意事項

- 同一 IP から短時間に失敗が集中した場合はセキュリティ確認を優先します。
- 障害中は一時的に告知バナーを表示します。

## FAQ

### Q. 再設定メールが届かない場合はどうしますか。

迷惑メール確認後、再送履歴と送信ドメイン設定を確認します。
